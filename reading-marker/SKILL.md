# 阅读标记

## 目标

把阅读材料标记成可复述的结构导航。标记不是装饰，不是把内容变醒目，而是让用户下次只看标记就能重建文章主线。

## 输入处理

- 使用用户提供的文章、笔记、转写稿、Markdown 文本或文件路径。
- 如果没有原文，先要求用户提供文本或文件路径，不要凭空生成标记。
- 如果用户提供了已有标记，默认保留原文表述，只调整标记；除非用户明确要求改写内容。
- 使用兼容 Markdown 的标记形式：`**关键词**`、`**<u>主线词</u>**`、`<mark>论点句</mark>`、`>` 引用块。

## 工作流程

1. 先用一句话写出文章主线。主线说不清时，不要大面积标记；直接说明不清楚的地方，并保守处理。
2. 按顺序标记：主线词、普通关键词、核心论点句、金句或故事。
3. 选择 3-5 个能串起全文逻辑的主线词，标为 `**<u>词语</u>**`。下划线不是“更重要的加粗”，只表示主线锚点。
4. 将概念名、方法名、关键术语、对比关系中的核心词标为 `**词语**`。
5. 将判断句、结论句、方法句标为 `<mark>句子</mark>`。背景色只用于句级论点，不用于单个词。
6. 将可复用金句、故事、案例、类比标为引用块。故事本身不要满身加粗或高亮，重点标故事推出的结论。
7. 第一轮标完后必须删减。删除那些“看起来不错”但不能帮助重建主线的标记。

## 硬规则

- 同一种样式只能表达一种含义。
- 加粗表示概念或关键词。
- 加粗加下划线只表示主线锚点。
- 背景色表示句级论点或判断。
- 引用块表示金句、故事、案例或类比。
- 不要整段高亮；段级材料用引用块。
- 每篇文章的主线锚点控制在 3-5 个。
- 每篇文章的背景色论点句控制在 5-8 句，除非用户明确要做密集学习笔记。
- 组合标记最多 3-5 处，只用于全文最核心的记忆钩子。
- 最终验收标准只有一个：只看标记，能不能复述文章主线。

## 输出格式

直接标记文本时，按以下格式输出：

```markdown
## 主线
[用一句话说明文章主线]

## 标记后文本
[保留原文结构并加入标记]

## 只看标记复述
[只基于标记复述文章骨架]

## 删减说明
[简要说明删掉或压低了哪些标记]
```

编辑文件时，保持原文结构，在原文件中加入标记，然后报告修改文件和校验结果。
