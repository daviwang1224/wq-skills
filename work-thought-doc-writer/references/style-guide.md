# 写作风格

## 基本取向

- 观点先行：开头直接给判断，不绕背景。
- 从现象抽象结构：不只描述事件，要提炼组织、产品、协作、认知或方法层面的结构性问题。
- 语言务实：少官话、少套话、少口号。
- 判断明确：可以锋利，但不能写成批判具体人的文章。
- 适合沉淀：文章应适合放入组织知识库、个人工作思考或团队认知材料。

## 判断表达

- 每篇文章只围绕一条主判断展开。
- 文章需要有可复述的核心判断或金句，但不能为了金句牺牲准确性。
- “不是 A，而是 B”是常用思维结构，不是标题硬模板。可以用于核心判断、段落句、小标题，也可以不用。
- 需要指出问题结构，不审判具体人，不把个案写成批判文章。

## 类比规则

- 可以使用历史、经济学、管理学、工程、医学、金融、体育等类比。
- 类比必须服务观点迁移，帮助读者把抽象判断理解清楚。
- 不为了显得有文化而堆理论、堆名词、堆外部案例。
- 类比不能引入用户未提供且无法核验的事实；不确定来源时必须说明不知道。

## 标题规则

标题不强制使用反差表达。优先级如下：

1. 言简意赅。
2. 中心思想明确。
3. 方便记忆。
4. 便于知识库检索。
5. 有判断感。

可选标题形式：

- 直接判断型：质量保障的价值在于防止问题复发
- 问题引导型：为什么越努力，复杂工作反而越乱
- 反差判断型：真正的分水岭不是努力，而是顺序
- 概念提炼型：阅读标记是一种认知导航

## 禁用倾向

- 不写官方套话、职场空话、口号式总结。
- 不写成情绪化批评。
- 不把文章写成清单大全。
- 不用理论压过素材。
- 不虚构冲突、后果、人物动机或组织背景。
