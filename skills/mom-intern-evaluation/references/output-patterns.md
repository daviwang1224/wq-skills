# MOM实习生绩效考核输出规范

## 报告结构

报告正文只包含以下四部分：

```markdown
# [yyyy-mm] MOM实习生绩效考核报告

## 一、实习生评分表
## 二、优秀实习生事迹
## 三、其他实习生综合评价
## 四、C档实习生改进建议
```

不要增加“团队整体评价”。

不要增加“质量检查”章节。

## 一、实习生评分表

必需内容：

- 固定10名实习生全部出现。
- 四维度评分、总分、评级。
- 表格按总分从高到低排序。
- 评级只允许A、B、C。

格式：

```markdown
| 序号 | 姓名 | 工作量(25) | 工作深度(25) | 工作成果(30) | 学习态度与协作(20) | 总分 | 评级 |
|------|------|------------|--------------|--------------|---------------------|------|------|
```

评分说明保留一句即可：

```markdown
**评分说明**：本评分为AI基于工作日志分析的建议，最终结果需负责人确认。
```

## 二、优秀实习生事迹

只写A档实习生。

如果没有A档，直接写：

```markdown
本月无A档实习生。
```

有A档时使用：

```markdown
### 姓名（A，X分）

- **突出表现**：基于日志写具体事实。
- **关键成果**：写可验证交付或闭环结果。
- **综合评价**：用1-2句话说明为什么达到优秀。
```

不要写空泛表扬。

## 三、其他实习生综合评价

只写B档实习生，不写A档和C档。

格式：

```markdown
| 序号 | 姓名 | 评级 | 综合评价 |
|------|------|------|----------|
```

综合评价控制在1-2句话，结构为：

事实表现 + 主要不足 + 下月建议。

## 四、C档实习生改进建议

固定写2名C档实习生。

格式：

```markdown
| 序号 | 姓名 | 总分 | 主要问题 | 改进要求 |
|------|------|------|----------|----------|
```

要求：

- 主要问题必须基于日志事实或数据缺失。
- 改进要求必须具体，例如补齐日志、提高任务闭环、加强反馈、明确每日产出。
- 不要使用委婉话掩盖不合格结论。

## 语言规则

- 直接、客观、可审核。
- 不写职场套话。
- 不编造日志中不存在的事实。
- 信息不足就写“日志未体现”或“缺少有效日志”，不要补故事。
- 删除所有TODO和HTML注释。
