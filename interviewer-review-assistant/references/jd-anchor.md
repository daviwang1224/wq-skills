# 团队专用 JD 锚点

## 使用方式

- 本文件只服务于当前团队的招聘标准，不是通用市场 JD，不要外推成行业普遍要求。
- 只在判断候选人岗位类型、岗位贴合度或能力覆盖完整性时读取本文件。
- 先识别岗位，再只使用对应岗位的专业要求，不要把所有岗位要求一起灌进上下文。
- 共性要求只保留高价值锚点，不重复展开教育背景、通用套话和重复软素质。
- 如果文本明显对应其他团队、其他级别或不同业务背景，本文件只能作为弱参考，并在报告里明确说明“不完全适配”。

## 通用要求

1. 候选人通常对应高级工程岗位，核心看点是是否能主导复杂模块、方案设计、工程化落地和高质量交付。
2. 应关注跨角色协作、带教能力、自驱推进和问题闭环，而不是只看单点执行。
3. 制造业软件、MOM、低代码平台经验属于优先项或加分项。
4. AI 工具熟练使用是硬门槛：
   - 前端、Java：强调用 Claude Code、Codex 等完成开发、重构、性能优化、复杂问题排查。
   - 测试：强调用 Cursor、Trae、Claude Code 等完成用例生成、问题分析、自动化脚本编写。
5. 如果面试全程没有验证 AI 协作能力，应直接指出验证缺口。
6. 如果本轮不是以深挖能力覆盖为目标，不要因为未覆盖全部锚点就机械判差，要结合面试轮次一起判断。

## 岗位专业要求

### 前端开发工程师（高级）

- 技术栈锚点：JavaScript、TypeScript、HTML5、CSS3、React 18+、UmiJS 4、Ant Design、Wujie、Vite、Webpack。
- 关键能力锚点：
  - 复杂业务模块开发
  - React 组件库封装
  - 前端项目框架搭建
  - 工程化与构建优化
  - 性能优化
  - 微前端架构设计与落地
- 提问应优先压到真实项目、技术取舍、架构方案和落地结果。

### 测试工程师（高级）

- 技术锚点：测试架构设计、测试策略、API/UI 自动化、Selenium、Pytest、RestAssured、CI/CD、环境部署、版本发布、基础运维监控、Python。
- 关键能力锚点：
  - 自动化测试框架搭建与复用
  - 复杂业务模块测试方案设计
  - 需求分析与评审参与深度
  - DevOps 流程落地
  - 测试资产沉淀
  - 问题闭环和质量改进
- 提问应优先压到测试方案、自动化落地、环境与发布、工程化能力和业务理解。

### Java 开发工程师（高级）

- 技术栈锚点：JDK 17、Spring Boot 3.1+、Spring Cloud Alibaba 2022.0.1、MyBatis、DM8、MySQL、Oracle、Maven、Gradle。
- 关键能力锚点：
  - 后端架构设计与优化
  - 复杂业务模块开发
  - 核心服务或工具类封装
  - 数据库设计、SQL 优化、事务处理
  - 微服务与分布式架构
  - 工程化和性能优化
- 提问应优先压到真实服务设计、架构取舍、数据库处理、性能瓶颈和工程化实践。
