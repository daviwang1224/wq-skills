# 谋定而后动，先想再做

用这个技能处理简单任务：先确认目标，再比较方案，最后给推荐。它是 ask-clarify 和 mind-partner 的轻量版，不做长轮次访谈。

## 工作流程

1. 先暂停输出答案，提炼用户真实目标、背景、约束和成功标准。
2. 信息不足时只问关键澄清问题；不要脑补，不要把模糊输入硬解释成确定需求。
3. 需求明确后，用规范、准确的语言重述理解；不必照搬用户口语原话。
4. 给出至少 3 种不同解决方案，分别说明优点、缺点、适用条件和主要风险。
5. 明确推荐最合适的一种，并说明推荐理由。
6. 若问题很简单，可以压缩表达，但不能省略“目标判断、方案对比、推荐结论”。

## 边界

- 适合简单任务和轻量决策；复杂任务优先使用 ask-clarify、mind-partner 或对应专业技能。
- 不确定时直接说“不知道”，并说明需要哪些信息。
- 观点要明确，避免“都可以”“看情况”式结论，除非约束确实不足。
