# 仓库维护说明

本仓库按通用 Agent Skills 规格维护。主力维护工具是 Codex，但技能文档不能写成 Codex 专属格式。

## 定位

- 目标是让技能的结构、元数据和说明格式尽量通用。
- 不要求技能内容变成通用业务模板。
- 业务技能可以保留具体领域、团队、流程、评分规则、个人画像和脚本逻辑。
- 仓库级文档不要绑定某个 Agent 客户端、插件市场或安装命令。

## Skill 基本结构

每个技能目录至少包含 `SKILL.md`：

```text
skill-name/
├── SKILL.md
├── scripts/
├── references/
├── assets/
└── agents/
```

只有 `SKILL.md` 是必需文件，其余目录按需保留。

## SKILL.md Frontmatter

优先使用最小通用字段：

```yaml
---
name: skill-name
description: 说明技能做什么，以及什么场景下使用。
---
```

要求：

- `name` 使用小写字母、数字和连字符。
- `description` 写清楚能力和触发场景。
- 不在 frontmatter 中写客户端专属工具列表。
- 不把“何时使用”只写在正文里，触发信息必须进入 `description`。

## 正文写法

- 只写执行技能所需的信息。
- 保持 `SKILL.md` 精简，详细规则放入 `references/`。
- 可重复且确定性的处理逻辑放入 `scripts/`。
- 输出格式、评分规则、业务口径可以保留在技能内。
- 不写面向某个客户端的安装、市场、菜单、命令说明。

## 资源目录

- `scripts/`：用于清洗、转换、生成模板等确定性任务。
- `references/`：用于放评分规则、输出规范、业务规则、示例模板。
- `assets/`：用于放会被复制或作为输出基础的模板、图片、样例文件。
- `agents/`：用于放特定 Agent 客户端的展示元数据；该目录可选，不应影响通用技能主体。

## 维护流程

1. 新增技能时，在 `skills/` 下创建同名目录。
2. 编写 `SKILL.md`，先保证 `name` 和 `description` 清晰。
3. 需要脚本时放入 `scripts/`，并保留可直接运行的用法说明。
4. 需要长规则时放入 `references/`，并在 `SKILL.md` 中说明何时读取。
5. 更新 [README.md](README.md) 的技能列表和数量。
6. 检查是否出现客户端专属定位、过期技能列表、错误链接和本机绝对路径。

## 检查清单

- [ ] 每个技能目录包含 `SKILL.md`。
- [ ] `SKILL.md` frontmatter 包含 `name` 和 `description`。
- [ ] 技能列表与实际目录一致。
- [ ] 仓库级文档不绑定某个 Agent 客户端。
- [ ] 技能格式说明与业务内容边界清楚。
- [ ] 脚本、引用资料和资产目录只在有实际用途时保留。
- [ ] 不新增与技能运行无关的冗余文档。

## 本仓库约定

- 全程使用中文维护说明。
- 读取或检索文件时优先使用 PowerShell 原生命令。
- 修改文件前先说明修改方案，获得确认后再执行。
- 不回滚他人已有改动。
